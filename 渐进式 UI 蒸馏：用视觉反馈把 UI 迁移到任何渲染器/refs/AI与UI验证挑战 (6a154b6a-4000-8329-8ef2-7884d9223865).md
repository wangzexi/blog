它追求的是：

“理解是什么”

“测量是多少”

## 第 6 轮 — ChatGPT

对，你遇到的是**反向提纯**问题。

生产代码里真实 UI 不是静态代码决定的，而是：

```text
最终视觉 = 样式代码
        + 运行时 props
        + 数据状态
        + 实验分支
        + 容器环境
        + 平台差异
        + 组件上下文
```

所以 AI 只从代码里搬，会不准。因为它搬的是“源码片段”，不是“视觉事实”。

更合理的流程应该是：

```text
生产页面
↓
运行时截图 + DOM/RN Tree + props + 样式快照
↓
AI 反向抽取视觉组件
↓
独立设计仓库复现
↓
截图 diff 校准
↓
设计师修改
↓
AI 应用回生产仓库
```

核心是：**不要让 AI 靠读代码猜 UI，要让它拿运行时事实校准 UI。**

你可以把这个叫：

```text
从生产 UI 中蒸馏 Design Runtime
```

或者：

```text
运行时反向抽取样式层
```

这里的关键工程点是：

1. **先选固定场景**

import numpy as np

def red_center(path):
    img = np.array(Image.open(path).convert("RGB"))

    r = img[:, :, 0]
    g = img[:, :, 1]
    b = img[:, :, 2]

    # 找大红色区域，可按你的 debug 色调整阈值
    mask = (r > 200) & (g < 80) & (b < 80)

    ys, xs = np.where(mask)

    if len(xs) == 0:
        raise ValueError("没有找到红色像素")

    return {
        "center_x": xs.mean(),
        "center_y": ys.mean(),
        "count": len(xs),
        "min_x": xs.min(),
        "max_x": xs.max(),
        "min_y": ys.min(),
        "max_y": ys.max(),
    }

target = red_center("target.png")
current = red_center("current.png")

dx = current["center_x"] - target["center_x"]
dy = current["center_y"] - target["center_y"]

print("target:", target)
print("current:", current)
print("dx:", dx, "dy:", dy)

## 第 30 轮 — ChatGPT

对，重心有用。

它的价值不是“精确描述边界”，而是提供一个**稳定的位置平均信号**。

不用重心，只看边界：

```text

```

会很容易抖。比如你改宽度时，右边界变了，系统可能误以为“位置偏了”。

所以最好分工：

```text

边界 dl/dr/dt/db：判断哪一边多了或少了
投影 dw/dh：判断整体宽高差
area/xor：判断最终重合度
```

你可以这样理解：

```text
重心 = 位置锚点
边界 = 形状诊断
投影 = 尺寸诊断
xor = 最终验收
```

调参顺序也应该固定：

```text
1. 先调尺寸：dw/dh 接近 0
2. 再调位置：dx/dy 接近 0

```

所以不是“重心没用”，而是**不能只用重心**。它适合判断整体平移，不适合判断宽高和形状。
